## 2024-05-24 - Integer Truncation in MIDI-CI SysEx Size
**Vulnerability:** The Maximum SysEx size (`maxSysex`) field in MIDI-CI was parsed as a 32-bit integer but stored and passed to callbacks as a 16-bit integer (`uint16_t`). This truncation (values > 65,535 bytes) could lead to buffer overflows or incorrect memory allocation if an attacker sends a SysEx payload corresponding to the larger original size.
**Learning:** The vulnerability existed because the internal storage array `intTemp` and the public callback signatures were defined with insufficient width (`uint16_t` instead of `uint32_t`) for a 28-bit MIDI-CI field.
**Prevention:** Always verify that internal storage types and callback signatures match the maximum bit-width defined by the protocol specification, particularly for size and length fields.
